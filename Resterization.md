# 正交投影与透视投影
在图形学中，正交投影和透视投影是两种将三维场景映射到二维观察平面上的基本方法，它们最核心的区别在于是否模拟人眼的“近大远小”的透视效果。具体而言，透视投影模拟了人眼或真实相机成像的物理过程，其视景体是一个平截头体（Frustum），即一个被裁剪掉顶部的棱锥。在这种投影下，距离视点越远的物体，投影到近平面上的尺寸就越小，从而使得平行线在无穷远处会看起来交汇于一点，形成消失点。这种非线性缩放（基于物体深度的倒数 1/z 的关系）带来了极强的立体感和纵深感，因此它被广泛用于3D游戏、虚拟现实等追求视觉真实感的场景。相比之下，正交投影则不模拟人眼视觉，其光线被视为平行传播且垂直于投影面，视景体是一个规则的立方体。在这种投影下，无论物体距离视点有多远，它在屏幕上投影后的大小都保持不变，平行线在投影后依然保持平行，没有消失点。这种线性缩放（通过直接平移和缩放实现）的特性使得物体的相对尺寸和角度被精确保留，因此正交投影主要应用于工程制图、计算机辅助设计（CAD）、策略游戏的顶视角，以及阴影贴图中生成方向光的深度图等对精确度和尺寸对比要求较高、而不需要真实立体效果的场景。

# MVP变换流水线
它分别代表**模型（Model）、视图（View）和投影（Projection）** 三个变换阶段。模型变换负责将物体自身的局部坐标转换到世界坐标，即决定物体在世界场景中的位置、旋转和缩放；视图变换则通过观察矩阵将世界坐标转换到相机空间（或观察空间），其作用相当于把相机移动到世界原点并调整好朝向，从而确定场景中哪些部分在相机的视野内；最后，投影变换（分为正交或透视）将相机空间内的坐标转换到裁剪空间，并通过后续的透视除法和视口变换映射到屏幕上的二维像素坐标。这一整套MVP矩阵链式相乘（最终结果为 P×V×M×局部坐标）保证了无论物体在场景中如何摆放、相机从哪个角度观察，最终都能正确地投影到屏幕上，从而实现三维场景的二维渲染。

# 什么是 Resterization？
光栅化是将以数学形式描述的几何图元（如点、线、三角形）转换为屏幕上由离散像素组成的图像的过程，它是图形渲染管线中从连续几何到离散像素的核心转换步骤。具体来说，给定三个经过顶点着色器变换和投影后的三角形顶点（在屏幕空间具有坐标 $(x_i, y_i, z_i)$ 及其相关属性），光栅化器需要确定该三角形覆盖了哪些像素（或更精确的采样点），并为每个覆盖的像素生成一个**片段**（Fragment）——片段包含该像素在三角形内的插值属性（如深度、法线、纹理坐标），后续将由片段着色器计算最终颜色。光栅化通常包含三个子阶段：**三角形设置**（计算三角形边方程和参数导数，用于后续的扫描转换）、**扫描转换**（使用重心坐标或边函数遍历像素并判断是否在三角形内部）、以及**属性插值**（根据重心坐标对顶点属性进行透视矫正插值）。由于三角形是凸多边形且屏幕像素是规则网格，判断一个像素是否在三角形内部的经典算法是**半平面测试**：对于三角形的三条边 $E_0, E_1, E_2$，分别计算像素中心点到每条边的符号距离，如果三个符号都为正（或负，取决于顶点顺序），则像素在三角形内部。现代 GPU 光栅化器使用 2x2 像素块（quad）进行并行测试，支持多重采样（MSAA）以及保守光栅化（Conservative Rasterization）等功能。光栅化相比光线追踪在硬件实现上极其高效，能保证每个时钟周期输出大量片段，是实时渲染（尤其是游戏、UI、AR/VR）的核心技术——它的“不够物理真实但足够快”特性，与光线追踪的“物理真实但慢”形成鲜明互补，现代混合渲染管线中也会先通过光栅化生成 G-Buffer，再在着色器中用光线追踪做局部光照或反射补充。

# 锯齿化/走样是怎么形成的？
在图形学中，锯齿（走样）的形成核心原因只有一个：**采样频率不足以捕捉信号的频率变化**。通俗来说，就是**像素不够细，导致连续的图形在离散的像素网格上被错误地呈现**。

具体形成过程可以这样理解：三维场景中的物体边缘在数学上是连续的直线或曲线。当这条直线经过屏幕的像素网格时，光栅化单元需要在每个像素中心（或覆盖区域）做一个采样决策——判断这个像素是应该显示为物体的颜色，还是背景的颜色。对于一条斜线来说，它不可能恰好完美地填满整列或整行像素，结果就是一部分像素被完全标记为物体颜色，相邻的像素被完全标记为背景颜色，两者之间缺少渐变的过渡。这些被“强制二值化”的像素点排列在一起，人眼就会看到原本光滑的斜边呈现出明显的阶梯状凸起，这就是我们常说的锯齿。

从信号处理的角度看，这是因为屏幕像素的采样率（即分辨率）低于图形信号中最高频率成分的两倍（奈奎斯特采样定律）。物体的尖锐边缘包含了极高的频率信息（从有到无的突变），而有限分辨率的屏幕无法忠实还原这种突变，于是高频信号在采样后“伪装”成了低频的阶梯状图案，也就是走样。

除了几何边缘的锯齿，走样还会以其他形式出现，例如：**纹理走样**（远处纹理闪烁或摩尔纹），这是因为纹理的细节频率超过了屏幕像素的采样频率；以及**运动走样**（车轮看起来倒转），这是因为时间上的采样率（帧率）不足以捕捉高速运动的轮辐。

# 抗锯齿/反走样的方法
在实际应用中，MSAA主要凭借其几何边缘锐利、不引入模糊的优点，用于对精度要求极高的静态场景，如CAD工程制图、产品静帧渲染，以及移动端和Forward渲染管线的游戏；FXAA则因其极低的性能开销而成为电子竞技游戏（如CS：GO）和低端硬件设备的首选，也常作为后处理链中兜底的抗锯齿方案；而TAA目前已成为绝大多数现代3A游戏（如《赛博朋克2077》及虚幻引擎5的作品）和主机游戏的主流选择，它能够以低成本稳定复杂场景中的植被、铁丝网等高频细节并消除闪烁，甚至构成了当今DLSS和FSR等超分辨率技术的底层基础。
### MSAA（Multiple Sampling AA 多重采样抗锯齿）
MSAA通过在每个像素内部增加多个覆盖采样点（如4x、8x），但对颜色计算只执行一次，仅在光栅化的覆盖测试阶段判断三角形盖住了多少个子采样点，若只盖住一部分则按比例混合背景与前景色，从而在几何边缘产生平滑过渡。其优点是相比SSAA性能开销小得多，能在保持较高帧率的同时有效消除几何边缘锯齿，且边缘保持清晰不引入模糊；缺点是它无法处理透明纹理贴图内部的Alpha Test锯齿，也不能解决像素着色器中的高频细节（如高光闪烁）产生的锯齿，且在现代延迟渲染管线中需要额外的硬件支持，内存占用会随采样倍数线性增长。
### FXAA（Fast Approximate AA 快速近似抗锯齿）
FXAA是一种纯粹的后处理技术，它不依赖任何几何信息或深度缓冲，而是直接对最终渲染的图像进行亮度分析，通过检测相邻像素间的亮度突变来识别边缘，然后根据边缘的方向和强度对周围像素进行加权模糊混合，从而平滑锯齿。其优点是性能极高且稳定，几乎没有额外的时间开销，能够一次性处理几何边缘、透明纹理甚至着色器产生的所有类型的走样，实现简单兼容性好；缺点是它是一种“猜测性”算法，容易将本应清晰的细小纹理或文字误判为锯齿而过度模糊，导致画面整体偏软且细节丢失，在静态画面下边缘可能出现轻微的“毛毛躁躁”感，缺乏原生高分辨率的锐利度。
### TAA（Temporal AA 时间性抗锯齿）
TAA的核心思想是将采样分散到时间维度上，每帧在像素内使用不同的子像素偏移位置进行采样，并借助运动向量将当前帧与历史帧的像素信息进行累积混合，从而在单帧极低的采样成本下实现相当于超高质量的多重采样效果。其优点是能以极小的性能开销高效消除几何边缘、透明纹理甚至着色器内部噪声引起的各类锯齿和闪烁，尤其擅长处理摩尔纹和动态细碎高光，是现代3A游戏和虚幻引擎5的主流方案；但其缺点是容易在快速运动物体边缘产生“鬼影”伪影，依赖精确的运动向量输入（一旦运动信息错误会导致画面撕裂），且由于需要累积多帧信息，画面会不可避免地带有柔和甚至偏软的视觉风格，切换镜头或场景时可能需要几帧时间才能重建稳定抗锯齿效果。
